# Changelog

Все заметные изменения проекта будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
а версии проекта следуют [Semantic Versioning](https://semver.org/lang/ru/).

## [Unreleased]

### Added

- Добавлен файл лицензии MIT.

### Changed

- README обновлён для развёртывания на чистом Ubuntu-сервере через официальный репозиторий Docker.
- Шаблон systemd переведён с прямого запуска Python на управление Docker Compose.
- Серверная инструкция создаёт `.env` с правами `600` и уточняет формат `GROUP_ID`.
- Инструкция по Telegram и VK токенам уточнена без привязки к сторонним сервисам.
- Для stdout-логов контейнера включена ротация Docker logging driver.

### Security

- Обновлены runtime-зависимости, включая `idna` 3.15 и `urllib3` 2.7.0, для устранения известных уязвимостей.
- В CI добавлен автоматический аудит Python-зависимостей через `pip-audit`.
- Базовый Docker-образ переведён на Alpine, чтобы исключить уязвимые неиспользуемые системные пакеты из Debian-образа.

## [1.1.0] - 2026-06-22

### Added

- Логирование одновременно в консоль и ротируемый файл.
- Настройки путей, уровня логирования, размера и количества архивов через переменные окружения.
- Dockerfile и Docker Compose для воспроизводимого развёртывания.
- Постоянные Docker volumes для файла состояния и логов.
- GitHub Actions для проверки Python-кода, Compose-конфигурации и сборки Docker-образа.
- Автоматическая публикация multi-architecture Docker-образов в GitHub Container Registry.

### Changed

- ID последнего поста сохраняется только после успешной отправки в Telegram.
- Добавлены таймаут запросов к VK API, проверка конфигурации и корректное завершение по системным сигналам.
- Файл состояния сохраняется атомарно, чтобы снизить риск его повреждения.
- Контейнер запускается от непривилегированного пользователя с файловой системой только для чтения.
- Docker Compose по умолчанию использует готовый образ из GHCR и поддерживает закрепление версии.

## [1.0.0] - 2026-06-21

### Added

- Получение новых постов из группы или страницы VK через VK API.
- Пересылка текста и фотографий в Telegram-канал.
- Поддержка публикаций с несколькими фотографиями.
- Сохранение ID последнего обработанного поста в `last_post.json`.
- Настройка токенов, идентификаторов и интервала проверки через `.env`.
- Шаблон systemd-сервиса для запуска и автоматического перезапуска бота.
